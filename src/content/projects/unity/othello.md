목표는 오목 구조를 최대한 활용하는 것이고 그 과정에서 여러 개선점을 찾아내는 것이다.

## 돌 착수 구현

기존의 오목 프로젝트에서 사용한 placeStone을 그대로 사용하였다.  
바뀐점은 돌이 오델로 돌에 맞게 사이즈가 약간 커졌다는 점 뿐이다.

<br>

## 착수 흐름 구현

이것도 기존의 오목 프로젝트와 다를 바 없어서 그대로 사용했다.

<br>

## 오델로봇 구현

오델로 봇의 경우 어디에 돌을 두면 가장 많은 돌을 뒤집을 수 있는지를 기준으로 점수판을 생성하고, 그것을 기준으로 오델로 봇이 돌을 둘 위치를 선정하는 방식으로 구현하였다.  
그렇기에 오목봇을 만들때보다 비교적 시간이 덜 걸렸던 것 같다.

<br>

오목봇때와 마찬가지로 GameManager는 현재 차례인 player(User 객체)의 isBot 값이 true이면 해당 플레이어는 오델로봇임을 인지하고 Bot 클래스의 메서드로부터 봇의 착수 위치를 받아와서 착수한다.
```c#
public int[] getBotStoneIdx(BoardManager board, int botNum, int otherNum) {}
```
이때 점수판 생성은 BoardManager의 메서드를 통해 반환받기 때문에 매개변수로도 BoardManager의 객체를 넘겨준다. 

<br>

## 게임 승리 판정
오델로는 서로가 돌을 더 이상 놓을 수 없을때까지 진행한 후 승패판정을 진행한다.  
따라서 매번 승패판정을 할 필요가 없기에 오목보다 더 간단했다.  
둘 다 더 이상 돌을 둘 수 없는 경우 승패판정을 내어서 결과를 UI로 표시한다.  
추가로 중간에 한쪽이 돌을 둘 수 없는 경우가 발생하기도 한다.  
그럴 경우 오델로 규칙에 따라 턴이 넘어가지 않도록 구현하였다.

<br>

## 돌 뒤집기 애니메이션
돌을 착수했을 때 상대편의 돌을 뒤집는 로직이 실행된다.  
원래는 모든 돌이 한번에 뒤집히지만 이것을 하나씩 차례대로 뒤집어지도록 만들고 싶어서 코루틴을 통해 구현했다.  
```c#
private IEnumerator stoneFlip(int id, int x, int y, bool isBlackStone){}
```

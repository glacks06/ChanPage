### React와 React의 특징

React는 NPM(Node Package Manager) 모듈중의 하나이다. 즉, 무언가를 편하게 하기 위해 혹은 어떠한 기능을 사용하기 위해 이용하는 라이브러리인 것이다.

그럼 React는 정확히 무엇일까?  
<br>

React는 사용자 인터페이스를 만들기 위해 사용되는 facebook에서 만든 자바스크립트 라이브러리이다. 이는 프론트엔드에서  UI를 다룰 때 사용하는 라이브러리임을 의미한다.

React를 이용하며 UI를 부분적으로 나누어서 모듈화를 시킬 수 있는 데, 이러한 모듈화된 UI를 '컴포넌트(Componente)'라고 부른다.  
컴포넌트들을 모아 전체적인 페이지를 구성하여서 레고 조립하기에 비유하는 경우도 있다.

이렇게 컴포넌트로 제작하여 만들 수 있다는 특징덕에 코드를 재사용하는 면에서 큰 장점을 가지고 있다.

가령 어떠한 UI구성이 반복적으로 들어가야 한다면 같은 코드를 난잡하게 여러번 사용하지 않아도 간단한 코드만으로 해당 컴포넌트를 이용할 수 있다는 예시가 있겠다. 더 나아가서 이렇게 반복적으로 들어간 같은 코드를 모듈 하나를 수정함으로써 전체에 적용시킬 수 있다.

그러나 React에서 이보다 더 두드러지는 특징은 페이지를 다시 불러오지 않아도 html파일이 수정되는 기능이다.

다시말해 화면에 어떠한 변화가 생겼을 때 HTML 요소를 전부 업데이트 하지 않고, 업데이트되어야 하는 일부분만 업데이트하는 기술이다. 이를 가능케 하는 것은 가상 DOM이다.  
<br>

### 가상 DOM?

DOM(Document Object Model)이란 직역하면 문서 객체 모델이다.

브라우저가 HTML 문서를 이해할 수 있게 만든 자료구조이며 DOM에서는 XML, HTML 문서의 각 항목을 계층으로 표현하여서 생성, 변형, 삭제할 수 있도록 한다.

주로 자바스크립트에서 요소를 선택, 수정, 추가, 제거를 할 때 사용한다.

나는 이를 '웹 페이지를 오브젝트 형태로 표현해서 자바스크립트에서 이용하기 쉽게 한다~'라고 이해했다.

이러한 DOM의 특징으로는 트리구조라는 점, 수정 시 바로 웹 페이지에 반영된다는 점(LIVE), 자바스크립트로 조작이 가능하다는 점이 있다.

따라서 웹 개발을 진행할 때 자바스크립트로 요소를 건든다면 DOM을 건드린다라고 생각하면 될 듯 하다.

그럼 React에서의 가상 DOM이란 무엇일까?

가상 DOM은 말 그대로 실제 DOM의 가상버전 즉, 복제품이다. 이 복제품을 이용하여 사용자가 변화시킨 내용을 가상 DOM에게 반영시키고 실제 DOM과 비교하여서 어떤 부분이 바뀌었는지 검사한 후 바뀐 일부분만 업데이트하는 방식으로 작동한다.

이를 이용하면 실제 DOM보다 가벼운 가상 DOM을 먼저 변화시킴으로써 연산을 줄이고, 최소한의 업데이트를 통해 빠른 로딩속도와 반응성 높은 UI를 구현할 수 있다.

결과적으로 새로고침 없이 페이지를 변경시킬 수 있는 싱글 페이지  애플리케이션 구현이 가능해진다.  
<br>

### 마무리

React는 이 밖에도 React Native와 같이 모바일 애플리케이션 프레임워크도 제공하기 때문에 굉장히 유용하다. 따라서 React의 정확한 개념과 활용을 잘 공부해두면 매우 좋을 듯 하다.

## 컴퓨터 시스템의 4가지 핵심 용어

컴퓨터 시스템은 계층 구조로 이루어져 있으며, 흔히 정부 조직에 비유할 수 있다.

| 용어 | 비유 | 역할 |
|---|---|---|
| 운영체제 (OS) | 국가 또는 정부 | 컴퓨터의 모든 하드웨어와 소프트웨어를 관리하는 총감독 시스템 (예: Windows, macOS) |
| 커널 (Kernel) | 대통령과 핵심 행정부 | 하드웨어와 직접 통신하고 제어하는 유일한 시스템 소프트웨어이자 운영체제의 핵심 두뇌 |
| 쉘 (Shell) | 명령어 통역관 (비서실) | 사용자의 명령(키보드 입력 등)을 해석하여 커널에 전달하는 역할 |
| 터미널 (Terminal) | 민원 창구 (전화기) | 사용자가 텍스트(명령어)를 입력하고 결과를 텍스트로 출력받는 CLI 실행 창 |

※ 커널(Kernel)의 정체: 커널은 하드웨어가 아니라 소프트웨어이다. 일반 응용 프로그램이 아닌 시스템 전체를 관리하는 가장 핵심적인 '시스템 소프트웨어'로, 하드웨어와 응용 소프트웨어 사이의 중재자 역할을 수행한다.

<br>

## 명령어 전달 과정 (터미널, 쉘, 커널의 상호 관계)

사용자가 터미널에 `ls` (파일 목록 보기) 명령을 입력했을 때의 작동 순서는 다음과 같다.

1. 사용자가 터미널 창에 `ls`를 입력한다.
2. 터미널은 이 텍스트를 쉘에게 전달한다.
3. 쉘은 `ls`를 "파일 목록을 보여줘"라는 명령으로 해석한다.
4. 쉘이 커널에게 파일 목록을 요청한다.
5. 커널이 하드웨어(저장장치)에서 데이터를 가져온다.
6. 커널이 쉘에게 데이터를 돌려준다.
7. 쉘이 받은 데이터를 터미널에 전달한다.
8. 터미널이 화면에 파일 목록을 출력한다.

## 시스템 부팅(Booting) 과정

컴퓨터 전원을 켜는 순간부터 바탕화면이 뜨기까지의 부팅 과정은 다음과 같은 순서로 진행된다.
<br>

1. 전원 ON 및 BIOS(UEFI) 실행: 메인보드의 BIOS가 먼저 실행되어 CPU, RAM 등 하드웨어를 기본 점검(POST)한다.
2. 부트 로더(Bootloader) 실행: BIOS가 저장장치에서 부트 로더를 찾아 실행하며, 부트 로더는 부팅할 운영체제를 결정한다.
3. 커널 로딩(Kernel Loading): 부트 로더가 저장장치에 있던 커널 파일을 찾아 메모리(RAM)로 불러오며, 시스템의 제어권이 커널로 넘어간다.
4. 시스템 초기화(Init Process): RAM에서 깨어난 커널이 `init` 프로세스를 실행시켜 그래픽 드라이버, 인터넷 등 핵심 서비스들을 차례로 실행한다.
5. 로그인 및 GUI 환경 제공: 모든 준비가 끝나면 사용자에게 바탕화면(GUI)이나 로그인 화면을 띄운다.
<br>

※ 부팅 시 OS와 커널의 관계: 부팅 과정에서 "OS를 로드한다"는 것은 OS 전체 패키지를 한 번에 불러오는 것이 아니다. 부트 로더가 OS의 심장인 커널 파일만 먼저 RAM에 올려 깨우고, 이후 깨어난 커널이 스스로 나머지 OS 구성요소들을 불러와 시스템을 완성시키는 구조이다.

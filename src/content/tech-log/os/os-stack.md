## 1. 🗂️ 세그먼트(Segment)의 개념과 구성

프로그램이 실행될 때 메모리 공간은 데이터의 성격과 용도에 따라 논리적인 구획으로 나뉘며, 이를 세그먼트(Segment)라고 한다. 이는 프로그램의 안전성(코드 보호)과 메모리 관리의 효율성을 높이기 위함이다.

| 영역 | 역할 | 특징 |
|---|---|---|
| 코드(Code/Text) | 기계어 명령어가 저장되는 공간 | 읽기 전용 |
| 데이터(Data) | 초기화된 전역 변수 및 정적 변수 저장 | - |
| BSS | 초기화되지 않은 전역 변수 및 정적 변수 저장 | 실행 시 0으로 채워짐 |
| 힙(Heap) | 사용자가 직접 관리하는 동적 할당 영역 | `malloc` 등으로 제어 |
| 스택(Stack) | 함수 호출과 관련된 임시 데이터 영역 | - |

<br>

## 2. 🥞 스택(Stack) 영역과 자료구조의 관계 및 어원

메모리의 스택 영역은 하드웨어적으로 LIFO(Last-In, First-Out, 후입선출) 방식의 스택 자료구조를 그대로 구현한 것이다. CPU의 스택 포인터(SP) 레지스터가 위아래로 움직이며 데이터를 쌓고(Push) 제거(Pop)하는 원리로 작동한다.



'스택'이라는 이름은 식탁 위에 접시를 쌓는 모습에서 유래했다. 함수가 호출되면 기존 데이터 위에 새로운 함수의 데이터를 쌓아 올리고, 가장 나중에 호출된 함수가 가장 먼저 끝나야(Pop) 그 밑에 있는 이전 함수가 다시 실행될 수 있는 구조이기 때문이다.

<br>

```text
[ 메모리 높은 주소 (High Address) ]
+-------------------------------+
|      One 함수의 스택 프레임       |
|   [ 지역 변수: temp, a ]        | <-- 1. One이 먼저 살고 있었음
+-------------------------------+
|  ★ 복귀 주소 (Return Address)    | <-- 2. Two를 부르면서 "끝나면 여기로 와!" 하고 
|     (One 함수 코드의 11번째 줄)    |      이 주소를 툭 던져 놓고 감 (Sandwich!)
+-------------------------------+
|      Two 함수의 스택 프레임       |
|   [ 매개 변수: b ]             | <-- 3. 그 위에 Two가 새로 집을 지음
|   [ 지역 변수: c ]             |
+-------------------------------+
               | 
               V 
[ 메모리 낮은 주소 (Low Address) ] (현재 스택 포인터 SP 위치)
```

<br>

## 3. 📦 스택 프레임(Stack Frame)의 역할과 장단점

함수가 호출될 때 복귀 주소, 지역 변수, 매개 변수 등을 하나로 묶어서 저장하는 단위를 '스택 프레임'이라고 한다.

| 구분 | 특징 | 상세 설명 |
|---|---|---|
| 장점 | 자동 메모리 관리 | 함수가 리턴하면 스택 포인터만 옮겨 데이터를 한 번에 날릴 수 있음 (`free` 불필요) |
| | 재귀 호출 지원 | 호출될 때마다 새로운 프레임을 쌓아 서로 간섭 없이 독립적 실행 가능 |
| | 문맥 보존 | 함수 종료 후 돌아갈 복귀 주소와 이전 상태를 완벽하게 기억함 |
| 단점 | 크기 제한 (Stack Overflow) | 힙에 비해 크기가 작아 너무 큰 배열이나 무한 재귀 시 오버플로우 발생 |
| | 짧은 생명 주기 | 함수 종료 시 데이터가 즉시 증발하여 함수 밖으로 데이터를 살려 보낼 수 없음 |

<br>

## 4. 📉 스택의 주소 할당 방향

스택은 높은 주소(High Address)에서 시작하여 낮은 주소(Low Address) 방향으로 자라난다. 즉, 데이터가 쌓일수록(Push) 주소값은 작아진다.

* **높은 주소 (Bottom):** 가장 먼저 실행된 함수(예: `main`)가 위치한다.
* **낮은 주소 (Top):** 가장 최근에 호출된 함수(현재 실행 중인 함수)가 위치한다.

※ 성장 방향의 이유: 힙(Heap) 영역이 낮은 주소에서 높은 주소로 자라기 때문에, 두 영역이 양끝에서 서로 마주 보고 자라게 하여 남는 메모리 공간을 효율적으로 공유하기 위함이다.

<br>

## 5. 📏 스택의 일반적인 메모리 할당 크기

스택의 할당 크기는 운영체제나 컴파일러 설정에 따라 다르지만, 수 GB에 달하는 힙 영역에 비하면 매우 작게 설정된다.

* **Windows (Visual Studio):** 보통 1MB
* **Linux (GCC):** 보통 8MB

※ 주의 사항: 스택의 크기는 매우 제한적이므로, `int arr[1000000]`과 같이 너무 큰 배열은 스택에 선언하지 말고 힙에 동적 할당해야 한다.

## 1960년대: 개념의 탄생 🌱

<br>

1967년 - 시뮬라 (Simula)
최초의 객체 지향 언어로 평가받는다.
노르웨이의 크리스텐 니가드와 올레 요한 달이 개발했다.
현실 세계의 시뮬레이션을 위해 객체(Object), 클래스(Class), 상속(Inheritance)이라는 핵심 개념을 처음으로 도입했다.

<br>

## 1970년대: 아이디어의 확산과 정제 💡

<br>

1972년 - 스몰토크 (Smalltalk)
제록스 PARC의 앨런 케이(Alan Kay)가 주도하여 개발했다.
'객체 지향 프로그래밍'이라는 용어를 널리 알리고 개념을 정립했다.
모든 것을 객체로 취급하는 '순수한' 객체 지향 언어의 형태를 보여주었다.
그래픽 사용자 인터페이스(GUI) 개발에 큰 영향을 미쳤다.

<br>

## 1980년대: 주류 시장으로의 진입 🚀

<br>

1983년 - C++
비야네 스트롭스트룹이 C언어에 객체 지향 기능을 추가하여 개발했다.
기존 C 프로그래머들이 OOP를 쉽게 접할 수 있게 하여 폭발적인 인기를 얻었다.
시스템 프로그래밍, 게임 개발 등 성능이 중요한 분야에서 표준으로 자리 잡았다.

<br>

1986년 - Objective-C
Smalltalk의 메시지 전달 방식을 C언어에 접목했다.
Apple이 인수하여 macOS와 iOS 앱 개발의 핵심 언어로 사용되었다.

<br>

## 1990년대: 웹의 시대와 플랫폼 독립성 🌐

<br>

1991년 - 파이썬 (Python)
귀도 반 로썸이 개발했으며, 쉬운 문법과 높은 생산성을 특징으로 한다.
객체 지향을 포함한 여러 프로그래밍 패러다임을 지원하며 현재까지 큰 사랑을 받고 있다.

<br>

1995년 - 자바 (Java)
썬 마이크로시스템즈에서 개발했다.
"한 번 작성하면, 어디서든 실행된다 (Write Once, Run Anywhere)"는 슬로건으로 플랫폼 독립성을 강조했다.
웹 애플리케이션과 엔터프라이즈 시스템 개발의 표준 언어로 부상했다.

<br>

## 2000년대 이후: 현대적인 OOP 언어의 등장 ✨

<br>

2001년 - C#
마이크로소프트가 Java에 대항하여 .NET 프레임워크와 함께 발표했다.
Java와 유사한 문법을 가지면서도 현대적인 기능들을 적극적으로 추가했다.

<br>

2014년 - 스위프트 (Swift)
Apple이 Objective-C를 대체하기 위해 발표한 현대적인 언어이다.
안전성과 성능, 간결한 문법을 강조한다.

<br>

2016년 - 코틀린 (Kotlin)
JetBrains에서 개발했으며, 2017년 구글이 안드로이드 공식 개발 언어로 채택했다.
Java와 100% 호환되면서도 더 간결하고 안전한 코드를 작성할 수 있다.

## 객체의 탄생과 발전: 시뮬라 vs. 스몰토크

모든 것은 1960년대, 현실 세계를 모의실험하기 위해 만들어진 **시뮬라(Simula)**에서 시작되었다. 시뮬라는 처음으로 **'객체(Object)'**라는 개념을 도입했지만, 이는 기존 언어에 추가된 강력한 **'도구'**에 가까웠다.
이후 1970년대, **스몰토크(Smalltalk)**는 이 개념을 **'철학'**으로 승화시켰다. "모든 것은 객체다"라는 원칙 아래, 숫자부터 문자, 클래스까지 모든 것을 동등한 객체로 취급하고 이들이 서로 **'메시지'**를 주고받으며 상호작용하는 순수한 객체 지향 세계를 구축했다.

<br>

## 개념의 구체화와 현실적 구현

스몰토크의 이상적인 '메시지 패싱' 철학은 현대 언어에서 **다형성(Polymorphism)**이라는 이름으로 계승되었다. 이는 런타임에 실제 객체 타입에 따라 올바른 메서드가 실행되는 **동적 디스패치(Dynamic Dispatch)**를 통해 구현된다.
이 기술의 핵심에는 **가상 메서드 테이블(Virtual Method Table, VTable)**이 있다. 컴파일러가 클래스별로 메서드의 주소가 담긴 '메뉴판(VTable)'을 미리 만들어두고, 각 객체는 이 메뉴판을 가리키는 숨겨진 포인터(vptr)를 가짐으로써, 런타임에 빠르고 효율적으로 올바른 메서드를 찾아갈 수 있다.

<br>

## 언어의 설계와 동작 원리

프로그래밍 언어는 **'설계도(언어 명세)'**와 그것을 실제로 구현한 **'제품(컴파일러, 라이브러 등)'**으로 나뉜다. 이 구현체는 크게 두 단계로 일한다.

컴파일 타임 (설계 단계): 코드를 기계어로 번역하며 타입 검사, 문법 오류 확인, 최적화(예: inline 확장) 등을 수행한다. 최신 C++의 constexpr처럼, 이 단계에서 일부 계산이나 객체 생성을 미리 끝내 성능을 극대화하기도 한다.
런타임 (실행 단계): 컴파일된 프로그램이 실제로 실행되는 단계이다. new 키워드를 만나면 힙(Heap) 메모리에 객체를 위한 공간을 할당하고 생성자를 호출하는 등, 실제적인 작업이 이루어진다.

<br>

## 현대 언어의 설계 철학과 트레이드오프

현대의 주류 언어들(C++, Java 등)은 시뮬라와 스몰토크의 철학을 절충한 형태이다.

메모리 관리: C/C++ 같은 언매니지드 언어는 프로그래머가 직접 메모리를(new/delete) 관리하여 최고의 성능을 추구하는 대신 메모리 누수 등의 위험을 감수한다. 반면 Java/Python 같은 매니지드 언어는 가비지 컬렉터가 메모리를 자동으로 관리하여 안정성을 높인다.
타입 시스템: Python/Java 같은 스트롱 타이핑 언어는 타입 규칙을 엄격하게 적용해 안정성을 추구하고, JavaScript 같은 위크 타이핑 언어는 타입을 유연하게 자동 변환하여 편의성을 높인다.
결론: 현대 주류 언어는 성능을 위해 기본 자료형을 허용하는 시뮬라의 하이브리드 구조를 채택하면서도, 유연성을 위해 스몰토크의 다형성 철학을 받아들인 실용적인 합의점이라고 볼 수 있다.

<br>

## C++의 핵심 기능들

이러한 개념들은 C++의 구체적인 기능들을 통해 구현된다.

* 다형성: 가상 함수(virtual function)
* 코드 가독성: 연산자 오버로딩, 기본 인수, // 한 줄 주석
* 안전성과 효율성: 참조(&), 개선된 자료형 검사, Type-safe 메모리 할당(new/delete)
* 템플릿: 제약조건(Constraints)

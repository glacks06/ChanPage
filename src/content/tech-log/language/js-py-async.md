## 비동기 프로그래밍의 핵심과 파이썬(asyncio)

비동기 프로그래밍은 주로 I/O 작업에서 발생하는 '기다리는 시간'을 낭비하지 않고 다른 작업을 동시에 처리하여 프로그램의 전체 효율을 높이는 방식이다.



파이썬은 `asyncio` 라이브러리를 기반으로 `async`와 `await` 키워드를 사용한다.  
코드 실행 중 `await`를 만나면(예: 네트워크 요청, 파일 읽기 등), 해당 작업이 끝날 때까지 멈춰서 기다리지 않고 제어권을 이벤트 루프(Event Loop)에 넘긴다.  
이벤트 루프는 그동안 대기하지 않고 실행 가능한 다른 작업을 찾아 효율적으로 실행한다.

_※ 주의: 파이썬은 동기 방식이 기본이므로, 비동기를 사용하려면 일반적인 `requests` 대신 `aiohttp`처럼 `asyncio`를 지원하는 전용 라이브러리를 명시적으로 선택해야만 한다._

<br>

## 자바스크립트(JS) vs 파이썬(Python) 비교

두 언어 모두 싱글 스레드 이벤트 루프 모델을 기반으로 하며 `async/await` 문법을 제공한다는 공통점이 있지만, 생태계의 기본 전제 조건이 다르다.

| 구분 | 자바스크립트 (JS) | 파이썬 (Python) |
|---|---|---|
| **기본 설계** | 비동기가 기본 (Default) | 동기가 기본 (Default) |
| **생태계 특징** | `fetch()`나 `setTimeout`처럼 브라우저와 Node.js의 거의 모든 I/O API가 Promise 기반의 비동기로 설계됨 | 동기 라이브러리와 비동기 라이브러리가 명확히 분리되어 있어 개발자의 명시적 선택(Opt-in)이 필수적임 |

## 자바스크립트 비동기 프로그래밍의 진화 역사

자바스크립트의 비동기 처리는 코드의 가독성과 개발 편의성을 높이는 방향으로 크게 3단계에 걸쳐 진화해 왔다.



* **1단계: 콜백 (Callback)**
비동기 작업이 끝난 뒤 실행할 함수(콜백)를 인자로 넘기는 고전적인 방식이다. 비동기 작업을 순차적으로 처리해야 할 때, 콜백 안에 또 다른 콜백이 깊게 중첩되는 일명 '콜백 지옥(Callback Hell)'이 발생하여 코드가 매우 복잡해지는 치명적인 문제가 있었다.

* **2단계: 프로미스 (Promise)**
비동기 작업의 '미래 상태(성공 또는 실패)'를 나타내는 객체를 도입했다. `.then()`과 `.catch()`를 사용해 작업이 끝난 후의 동작을 체인(Chain)처럼 연결함으로써 콜백 지옥 문제를 우아하게 해결했다.

* **3단계: 최신 문법 (async / await)**
Promise를 훨씬 더 직관적이고 쉽게 사용하기 위해 도입된 '문법적 설탕(Syntactic Sugar)'이다. `await` 키워드를 사용해 Promise가 완료될 때까지 기다리도록 지시함으로써, 복잡한 비동기 코드를 마치 동기 코드처럼 순차적이고 읽기 쉽게 작성할 수 있게 해준다.

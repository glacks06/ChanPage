쌓임 맥락에 대한 궁금증에서 시작된 인공지능과의 질의응답 내용을 정리한 글이다.


<br>

## 1. 🔍 position: relative 요소가 float 요소를 가리는 이유
- Q: `position: relative` 요소가 `float` 요소를 가리는 이유는 무엇인가?
- A: `position: relative` 속성이 적용된 요소는 새로운 '쌓임 맥락'이라는 독립적인 3D 층을 생성하기 때문이다. 반면 `float` 요소는 새로운 쌓임 맥락을 만들지 않으므로, 기본적으로 층이 분리되어 `relative` 요소가 그 위에 렌더링된다.

<br>

## 2. 🥞 새로운 쌓임 맥락의 의미
- Q: '새로운 쌓임 맥락을 만든다'는 것은 무슨 의미인가?
- A: 특정 CSS 속성을 가진 요소가 다른 요소들과 완전히 독립된 3차원 공간의 층을 생성하는 것을 의미한다. 이 층은 마치 투명한 새로운 그림판처럼 동작하며, 이 그림판 내부에서만 `z-index` 순서가 유효해진다.

<br>

## 3. 🔒 z-index 순서의 독립성
- Q: '자식 요소는 그 층 안에서만 `z-index` 순서를 가진다'는 것은 어떤 의미인가?
- A: 새로운 쌓임 맥락이 생성되면, 그 안에 있는 요소들의 렌더링 순위는 오직 해당 쌓임 맥락 내에서만 의미를 가진다는 뜻이다. 부모 쌓임 맥락 밖의 요소와는 `z-index` 값으로 순서를 비교할 수 없으며, 자식 요소의 화면 위계는 전적으로 부모 쌓임 맥락의 위계에 종속된다.

<br>

## 4. 🥇 relative vs 일반 요소의 렌더링 순서
- Q: 일반적인 요소가 `z-index` 값을 아무리 크게 가져도 `relative` 요소를 가릴 수 없는 이유는 무엇인가?
- A: `relative` 요소는 새로운 쌓임 맥락을 생성하여 기본 층 위로 올라오지만, 일반 요소는 기본 층에 그대로 머무르기 때문이다. 이 둘은 서로 존재하는 층 자체가 다르므로 단순한 `z-index` 값 조정만으로는 렌더링 순서를 뒤집을 수 없다.

<br>

## 5. ✂️ z-index의 분리
- Q: 일반적인 요소의 `z-index`와 새로운 쌓임 맥락을 가진 요소의 `z-index`를 분리해서 생각해야 하는가?
- A: 그렇다. 기본 층에 있는 일반적인 요소의 `z-index`는 효과가 없으므로 무시하는 것이 맞다. 위계를 조정할 때는 새로운 쌓임 맥락을 생성한 요소들 사이에서만 `z-index`를 기준으로 순위를 결정해야 한다.

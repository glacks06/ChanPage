## SCSS에서 리스트(list)란?

SCSS에는 자바스크립트의 배열(Array)처럼 생긴 게 없다.  
대신 리스트(List)라는 개념이 배열 역할을 담당한다.

```scss
$colors: (red, green, blue);
```

이렇게 괄호 안에 값들을 나열하면,  
CSS 스타일을 그룹화해서 효율적으로 관리할 수 있는 리스트가 만들어진다.  
SCSS는 CSS 전처리기라서 복잡한 조작보다는,  
스타일을 편하게 묶고 컴파일하기 쉽게 만드는 게 목적이다.

<br>

## 리스트의 불변성 - “값은 절대 직접 못 바꿈!”

SCSS 리스트가 특이한 점은  
배열의 값을 직접 바꾸는 문법이 없다는 것.

```scss
// 이런 건 절대 불가능!
$colors[0] = yellow;
```

<br>

대신, 값을 바꾸거나 추가하려면  
기존 리스트를 복사해서 새로운 리스트를 만드는 함수를 써야 한다.

```scss
$colors: (red, green, blue);
$new-colors: list.set-nth($colors, 1, yellow); // 인덱스 1의 값을 yellow로 교체
```

즉, “원본 값은 보존, 새 리스트 만들어 쓰기”가 원칙이다.

<br>

## SCSS가 불변성을 고집하는 이유

<br>

**1\. CSS 특성**  
SCSS는 결국 선언적 언어인 CSS로 컴파일된다.  
CSS는 “지금 값을 바꿔라”가 아니라 “이렇게 스타일링해라”라는 목표라  
복잡한 상태 변동이 전혀 없다.

<br>

  
**2\. 예측 가능한 코드**  
불변성을 지키면  
같은 리스트를 여러 곳에서 써도, 갑작스런 값 변경(side effect)이 생기지 않는다.  
어떤 함수에서 값을 바꿨더니 해당 값에 의존하던 다른 프로퍼티가 있어서 버그가 발생하는 것과 같은 참사를 줄여준다.

<br>

  
**3\. 더 직관적인 의도 표현**  
“새 리스트를 만든다”는 패턴이  
실제 코드 의도를 더 명확하게 보여준다.  
상태를 바꾸려고 억지로 리스트 값을 바꾸는 대신,  
“새로운 값 집합을 만들었어!“라고 선언하는 방식이다.

<br>

## 마무리

scss의 리스트를 사용하려는데, 인덱스별로 값을 변경할 수 없다는 사실을 듯고 의아했었다.

하지만 css에서는 기존 리스트의 값 조작이 거의 필요가 없다는 것을 알게되었고, 여러 데이터 값들을 한가지 식별자로 다루기 위함이라는 것을 깨닫게 되었다.

## @layer

간단히 말하면 \`@layer\`는 스타일을 역할별로 그룹화해서 우선순위를 명확히 관리할 수 있는 CSS at-rule다.

  
• 테마 스타일이 다른 스타일에 덮여버리는 경우  
• 여러 CSS 파일을 쓰면서 충돌이 잦은 경우  
이걸 \`@layer\`가 해결해 준다.

<br>

## @layer의 사용법

\`@layer\`는 크게 두 가지 방법이 있다.

<br>

**1. 레이어 미리 선언하기**

```css
@layer reset, base, theme;
```

여기서 \`reset\`, \`base\`, \`theme\` 세 개의 레이어를 선언하고,  
선언한 순서가 바로 우선순위 순서다.  
즉, \`theme\` 레이어가 가장 높고, \`reset\`이 가장 낮다.

<br>

**2. 레이어에 스타일 묶기**

```css
@layer reset {
  * {
    margin: 0;
    padding: 0;
  }
}

@layer theme {
  .button {
    background-color: blue;
    color: white;
  }
}
```

<br>

## 외부 CSS 파일과 함께 쓰기

외부 CSS 파일도 특정 레이어에 포함시킬 수 있다.

```css
@import url('base.css') layer(base);
```

이렇게만 해도 \`base\` 레이어가 자동 생성되고, 내부 스타일과 우선순위를 함께 관리할 수 있다.  
단, \`@import\` 문법은 CSS 파일 맨 위에 위치해야 하며,  
중복 레이어 선언은 피해야 한다.

<br>

## 활용

• 모든 프로젝트 기본은 \`reset / normalize\`, \`base\`,  'components / modules', \`theme\` 등의 레이어 선언

-   **reset 또는 normalize**: 모든 브라우저의 기본 스타일(여백, 폰트 크기 등)을 일관성 있게 초기화하는 역할을 한다. **reset** 은 브라우저의 모든 기본 스타일을 제거해 백지상태에서 시작하고, **normalize는** 브라우저 간의 불일치만 수정하여 기본 스타일은 유지한다.
-   **base**: 사이트 전체에 공통적으로 적용되는 기본 스타일을 정의한다. body, h1, p, a, button 등과 같은 **HTML 태그 자체에 대한 기본 스타일**이 여기에 포함된다. 예를 들어, 모든 문단의 기본 폰트 크기나 링크의 색상 등을 지정하는 것이 있다.
-   **components 또는 modules**: 재사용 가능한 UI 요소, 즉 **컴포넌트의 스타일**을 정의하는 레이어이다. 버튼, 카드, 내비게이션 바, 폼 입력창 등과 같이 특정 기능을 가진 개별적인 요소들의 스타일이 여기에 속한다.
-   **theme**: 사이트의 전체적인 디자인 테마(색상 팔레트, 폰트 패밀리 등)나, 사용자의 설정에 따라 달라지는 **커스텀 스타일**을 정의하는 레이어이다. 다크 모드/라이트 모드 같은 기능이 여기에 해당한다.

<br>

## 마무리

스타일 간 충돌을 예방하고 관리하는 것과 관련한 명령어와 자료가 많은 것으로 미루어 보아 이는 매우 중요한 문제임을 알 수 있었다.

이번 @layer도 그중 하나로 한 번은 정리해 두는 것이 좋다고 생각했다.

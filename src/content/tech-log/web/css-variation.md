## CSS 변수

CSS 변수(커스텀 프로퍼티)는 \`--변수명: 값\` 형태로 선언하고, \`var(--변수명)\`으로 참조한다.

```css
:root {
  --primary-color: #3498db;
  --box-width: 500px;
}

.box {
  background-color: var(--primary-color);
  width: var(--box-width);
}
```

여기서 \`:root\`는 문서의 최상위 요소인 \`<html>\`을 가리킨다.  
즉, 전역 변수처럼 어디서든 참조할 수 있는 변수를 만드는 거다.

<br>

## 그런데 이 변수들은 어디에 저장되는 걸까?

처음에는 이런 생각을 했다. “JavaScript 변수처럼 메모리 어딘가에 저장되는 건가?”  
알아보니 CSS 변수는 CSSOM(CSS Object Model) 내 각 요소의 스타일 속성에 key-value 형태로 저장된다.

```css
/* 이렇게 선언하면... */
:root {
  --main-color: red;
}

/* 브라우저 내부에서는 대략 이런 식으로 관리 */
html: {
  customProperties: {
    "--main-color": "red"
  }
}
```

그리고 \`var(--main-color)\`를 만날 때마다 이 저장된 값을 찾아서 사용한다.

<br>

## 내부 처리 관점과 렌더링 관점

여기서 흥미로운 점을 발견했다. CSS 변수의 처리 방식이 관점에 따라 다르다는 것이다.  
렌더링 관점: 화면에 그릴 때는 변수명이 완전히 사라지고 실제 값으로 치환된다.  
내부 처리 관점: CSSOM이나 스타일 계산 시스템에서는 변수명과 참조 관계가 그대로 유지된다.

```css
.box {
  width: var(--box-width); /* 내부적으로는 이 참조 관계를 유지 */
}
```

<br>

```css
<!-- 실제 렌더링 시에는 -->
.box {
  width: 500px; /* 이렇게 값으로 치환되어 화면에 그려짐 */
}
```

이런 구조 덕분에 변수 값을 바꿔도 참조 관계는 그대로 유지되면서, 동적인 스타일 변경이 가능한 거다.

<br>

## JavaScript로 변수 값을 바꿀 때 일어나는 일

그럼 이런 코드를 실행하면 내부적으로 어떤 일이 벌어질까?

```javascript
document.documentElement.style.setProperty('--box-width', '600px');
```

조사해 보니 브라우저는 이런 과정을 거친다:

<br>

1. 값 변경 트리거  
    • \`--box-width\` 변수의 값이 \`500px\`에서 \`600px\`로 변경됨  
2. 의존성 그래프 탐색  
    • 브라우저가 어떤 요소들이 해당 변수를 사용하고 있는지 탐색함  
    • \`var(--box-width)\`를 쓰는 모든 요소를 파악  
3. 부분적 스타일 재계산  
    • 영향받는 요소들만 스타일을 다시 계산  
    • 나머지 요소들은 건드리지 않음  
4. 렌더 트리 갱신  
    • 필요한 부분만 리플로우/리페인트 수행

<br>

_※ 리플로우(Reflow)와 리페인트(Repaint)_

_리플로우(Reflow): 요소의 크기나 위치가 바뀔 때 레이아웃을 다시 계산하는 과정_  
_리페인트(Repaint): 계산된 결과를 바탕으로 화면에 다시 그리는 과정_  
_간단히 말해서,_  
_• 리플로우 = “어디에 배치할지 다시 계산”_  
_• 리페인트 = “실제로 화면에 그리기”_

<br>

## 의존성 그래프

이 과정에서 가장 중요한 건 의존성 그래프라는 개념이었다.  
브라우저는 각 CSS 변수와 이를 참조하는 모든 스타일 속성, 요소 간의 관계를 그래프 형태로 관리한다.

```css
--primary-color
├── .header { color: var(--primary-color) }
├── .button { background: var(--primary-color) }
└── .link { border-color: var(--primary-color) }
```

이렇게 관계를 미리 파악해 두기 때문에, 변수 값이 바뀔 때 전체 스타일시트를 다시 계산하지 않고도 영향받는 부분만 빠르게 찾아낼 수 있다.

<br>

## CSS 변수 vs 일반 CSS 속성

| 구분 | 일반 CSS | CSS 변수 |
| --- | --- | --- |
| 저장 방식 | 각 요소의 스타일 속성에 직접 저장 | key-value로 참조 관계 유지 |
| 변경 시 영향 | 해당 요소만 | 변수를 참조하는 모든 요소 |
| 동적 변경 | JavaScript로 각 요소별로 변경 | 변수 하나만 바꾸면 전체 적용 |
| 성능 | 요소별 개별 처리 | 의존성 그래프로 효율적 처리 |

## 마무리

## Native CSS

네이티브 CSS란, 브라우저가 전처리기 없이 바로 해석하고 실행하는 최신 CSS 표준 문법과 기능을 뜻한다.  
예전엔 Sass 같은 도구가 없으면 중첩, 변수, 함수 등을 직접 쓰기 어려웠다.  
하지만 현대 브라우저 엔진은 이런 기능을 그냥 지원해 준다.  
즉, 빌드나 컴파일 과정 없이 바로 CSS 파일을 브라우저가 읽고 스타일링한다는 거다.

<br>

## 네이티브 CSS의 핵심 개념

• 브라우저 직접 처리: 별도의 도구 없이 CSS 파일을 바로 읽고 적용한다.  
• 최신 문법 지원: CSS 변수(\`--변수명\`), 함수형 셀렉터, 중첩(\`:is()\`, \`:where()\`), 커스텀 속성 등  
과거엔 복잡한 스타일을 만들어내기 힘들던 기능들을 정식 표준으로 제공한다.

• 동적 스타일링 가능: CSS 변수는 JavaScript에서 조작할 수 있어서, 페이지가 로드된 후에도  
테마를 바꾸거나 사용자 인터랙션에 따른 스타일 변화를 실시간으로 줄 수 있다.

<br>

## Native CSS가 주는 장점

• 개발 환경 단순화 : 빌드 도구 설정이나 번거로운 전처리 과정을 없애고, 순수 CSS만으로 작업 가능하다.  
• 유지보수·가독성 향상: 중복된 코드가 줄고 문법이 간결해져서, 큰 프로젝트일수록 관리가 편해진다.  
• 런타임 스타일 변경 지원: JavaScript와 CSS가 훨씬 자연스럽게 연결돼, 유연한 UI 구현이 쉬워진다.  
• 브라우저 최적화: 브라우저 내부 파서가 최신 문법을 효율적으로 처리해 빠르고 안정적인 렌더링을 보장한다.

<br>

scss를 사용할 경우 아무리 변수명을 지정하고 이를 사용한다고 해도 이를 컴파일하는 과정에서 상수값으로 css에 고정된다.

즉, 컴파일을 한 순간부터는 '변수'라고 부를 수 없다는 것이다.

하지만 변수를 css에서 지원해준다면, 런타임 상황에서도 javascript를 통해 특정 요소의 스타일을 변경시킬 수 있다.

그렇다고 scss와 native css 파일을 각각 분리해서 작성해야만 하는 것은 아니다.

scss에서 컴파일을 하면 css의 형식에 맞게 텍스트를 재작성해주는 것과 다름이 없으므로 scss 파일에서도 얼마든지 native css 문법을 사용할 수 있다.

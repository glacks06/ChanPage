<br>

흔히 CSS 레이아웃을 구성할 때 `float`이나 `margin`을 조작하다 보면, 요소가 겹치거나 부모 요소의 높이가 사라지는 알 수 없는 현상을 겪곤 한다.
평소처럼 스타일을 조정하다가 문득 떠오른 생각이 있었다.
'왜 부모 요소는 `float`된 자식의 높이를 인식하지 못하고, `overflow: hidden`을 주면 갑자기 해결되는 걸까?'
이러한 궁금증을 계기로 조사해 보았다.

결론부터 말하자면 이는 CSS의 'BFC(Block Formatting Context)'라는 논리적인 레이아웃 격리 규칙 때문이며, 현대 웹 개발에서는 `flex`나 `grid`를 사용하여 이러한 복잡한 문제를 원천적으로 해결한다.

<br>

## 브라우저 스타일 통일 (시작점)

브라우저마다 기본적으로 적용하는 스타일이 다르기 때문에, 본격적인 레이아웃 구성 전 이를 통일하는 과정이 필요하다.

| 스타일 종류 | 특징 | 목적 |
|---|---|---|
| User Agent Stylesheet | 브라우저(크롬, 사파리 등)가 기본적으로 적용하는 CSS (`h1`의 크기 등) | 브라우저 제조사가 정한 기본 포맷 제공 |
| Reset CSS | 모든 태그의 여백, 글자 크기 등 기본 스타일을 0으로 강제 초기화 | 환경의 차이를 없애고 완전히 '백지' 상태에서 시작 |
| Normalize CSS | 모든 스타일을 제거하지 않고, 브라우저 간 차이점만 수정 | 유용한 기본값은 유지하면서 파편화 방지 |

※ CDN (Content Delivery Network): 이미지나 영상 등 무거운 파일을 사용자와 가장 가까운 '복사본 서버(캐시 서버)'에서 전송받게 하는 기술이다. 이를 통해 웹사이트 로딩 속도를 높이고 원본 서버의 부하를 줄인다. (마치 본점과 동네 편의점의 관계와 같다.)

<br>

## float과 전통적인 레이아웃 문제

과거에 `float: left`는 요소를 가로로 배치하는 거의 유일한 수단이었다.
하지만 자식 요소에 `float`을 적용하면, 자식이 '일반 흐름(Normal Flow)'에서 벗어나 공중에 뜬 상태가 된다.

이때 부모 요소는 공중에 뜬 자식의 높이를 전혀 인식하지 못해 자신의 높이가 0이 되어버리는 '높이 붕괴' 현상이 발생한다.

<br>

## BFC (블록 서식 컨텍스트) - 모든 문제의 해결책

`float`과 `margin` 문제를 해결하는 핵심 개념이 바로 BFC이다.

_※ BFC (Block Formatting Context): '독립된 레이아웃 환경'을 생성하는 CSS의 논리적인 규칙이다. BFC가 된 요소는 내부와 외부가 완전히 격리된다._

BFC는 `display: flow-root`(가장 권장), `overflow: hidden`, `position: absolute`, `display: inline-block` 등을 통해 생성할 수 있다. 
BFC의 핵심 기능은 다음과 같다.

* **float 감싸기 (Containment):** 부모 요소가 BFC가 되면, 공중에 뜬 자식 요소들의 높이를 인식하여 그 바닥까지 자신의 높이를 늘려 감싸 안는다. (높이 붕괴 해결)
* **마진 겹침 방지 (Isolation):** 일반적인 형제 요소는 같은 BFC에 속해 마진 겹침이 발생한다. 하지만 한 요소를 새로운 BFC 안에 넣으면 서로 다른 BFC에 속하게 되어 마진 겹침이 방지된다.

<br>

## 경계의 차이: BFC vs Box Model

| 경계 종류 | 설명 | 특징 |
|---|---|---|
| 박스 모델 경계 | 요소가 가진 `border`, `padding` 등을 포함한 크기 | 물리적인 공간 경계 |
| 레이아웃 경계 (BFC) | BFC가 생성되었을 때만 발생하는 격리 경계 | `float`을 가두고 `margin`을 격리하는 논리적 규칙 |

<br>

## BFC의 예외와 overflow: hidden의 이중성

`position: absolute`는 문서 흐름을 완전히 탈출하기 때문에 BFC의 감싸기 규칙이나 격리 규칙을 모두 무시한다.   
대신 `position: static`이 아닌 가장 가까운 조상(Containing Block)을 기준으로 배치된다.

여기서 `overflow: hidden`의 이중성이 나타난다. 
이 속성은 (A) 레이아웃 규칙에 영향을 주어 BFC를 생성하는 역할과, (B) 자신의 박스 모델 경계를 넘어가는 자식을 물리적으로 잘라내는(콘텐츠 숨기기) 역할을 동시에 수행한다.

`absolute` 자식을 숨기려면 부모가 (B) 기능을 수행해야 하지만, 자식이 부모의 BFC(A) 규칙을 무시하므로 잘라낼 수 없다.  
따라서 부모 요소에 `position: relative`를 주어 자식의 Containing Block으로 만들어야만 물리적으로 잘라내는 규칙에 영향을 받게 된다.

<br>

## 현대적인 해결책 (Flex & Grid)

이처럼 `float`과 BFC는 매우 복잡하고 혼란스럽기 때문에 현대 웹 개발에서는 레이아웃용으로 거의 사용하지 않는다.  
대신 `display: flex` 또는 `display: grid`를 주로 사용한다.

이들은 FFC/GFC라는 자신만의 독립적인 서식 컨텍스트를 생성하며, 여기서는 마진 겹침이 애초에 발생하지 않는다.  
또한 `margin` 대신 `gap` 속성을 사용하면 겹침 현상 없이 요소 간 간격을 매우 깔끔하게 제어할 수 있다.

_※ Tailwind CSS 같은 최신 유틸리티 라이브러리들은 모든 기본 마진을 0으로 초기화하고, `gap`이나 `space-y-` 같은 특수한 유틸리티 클래스를 사용하여 마진 겹침 문제가 아예 발생하지 않도록 설계되어 있다._

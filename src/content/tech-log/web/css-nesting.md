## CSS 네스팅

CSS 네스팅은 스타일 선택자를 계층적으로 중첩해서 작성하는 기능이다.  
HTML의 구조처럼 CSS도 중첩해서 쓸 수 있게 해주는 거다.

```css
/* 네스팅 방식 */
.card {
  padding: 20px;
  
  .card-title {
    font-size: 24px;
    color: #333;
  }
  
  .card-content {
    margin-top: 10px;
    line-height: 1.6;
  }
}
```

이렇게 쓰면 HTML 구조와 CSS 구조가 비슷해져서 훨씬 직관적이다.

다시 말해, 가독성이 올라간다.

_※ Nesting은 '둥지를 틀다'라는 의미가 있다. 부모 선택자 안에 자식 선택자가 둥지를 튼 것과 비슷한 모습이다._

<br>

## 브라우저의 네스팅 처리 방식

처음에는 이런 생각이 들었다. “브라우저가 중첩된 CSS를 직접 이해하는 건가?”  
알아보니 그렇지 않다.  
네스팅 문법은 결국 평면화(flattening) 과정을 거쳐서 기존 CSS 형태로 변환된다.

```css
/* 네스팅으로 작성한 코드 */
.card {
  padding: 20px;
  
  .card-title {
    font-size: 24px;
  }
}
```

<br>

```css
/* 평면화된 결과 */
.card {
  padding: 20px;
}

.card .card-title {
  font-size: 24px;
}
```

즉, 브라우저는 결국 평면적인 CSS 선택자로 변환된 것을 처리한다.

<br>

## 전처리기 네스팅 vs 네이티브 CSS 네스팅

네스팅을 구현하는 방법은 크게 두 가지가 있다.

<br>

  
1. 전처리기 방식 (Sass, Less 등)  
• 개발 시 컴파일 단계에서 평면 CSS로 변환  
• 빌드 도구가 필요하고 설정도 해야 함  
• 브라우저에는 이미 변환된 CSS가 전달됨

<br>
  
2. 네이티브 CSS 네스팅  
• 브라우저 내부에서 런타임에 평면화  
• 별도 빌드 도구 없이 바로 사용 가능  
• 최신 브라우저가 직접 지원

다른 도구 없이 순수히 css를 쓴다면 Native CSS의 네스팅을 쓰게 될 것이다.

<br>

## 네스팅 처리 흐름

그럼 네스팅이 실제로 어떤 과정을 거쳐 처리되는지 보자.  

<br>

1. 중첩 CSS 작성 개발자가 네스팅 문법으로 스타일 작성  
2. 평면화 과정  
• 전처리기: 빌드 시 일반 CSS로 변환  
• 네이티브: 브라우저 파서가 동적으로 평면화  
3. 스타일 계산 및 매칭 평면화된 선택자로 DOM 요소와 매칭  
4. 렌더 트리 생성 및 페인팅 기존과 동일한 렌더링 과정

<br>

## 네스팅 기능의 핵심

네스팅 기능의 핵심을 한 줄로 정리하면:  
“작성은 중첩으로, 처리는 평면으로”  
브라우저의 근본적인 CSS 처리 방식은 그대로이고, 단지 개발자가 더 직관적으로 코드를 작성할 수 있게 해주는 문법적 편의 기능인 셈이다.

```css
/* 개발자가 쓰는 방식 (직관적) */
.header {
  background: white;
  
  .nav {
    display: flex;
    
    .nav-item {
      padding: 10px;
    }
  }
}
```

<br>

```css
/* 브라우저가 처리하는 방식 (평면적) */
.header { background: white; }
.header .nav { display: flex; }
.header .nav .nav-item { padding: 10px; }
```

<br>

## 마무리

scss를 사용해 보면서 네스팅 기능 덕분에 가독성이 많이 올라감을 느꼈었다.

이러한 기능을 기본 css에서도 사용할 수 있어서 편리하다는 생각이 든다.

변화하는 기술들에 맞게 더 편리한 방식을 잘 채택하여 적절히 사용한다면 굉장한 도움을 얻을 수 있을 것이다.

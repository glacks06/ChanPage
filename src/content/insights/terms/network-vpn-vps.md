## 홈 호스팅 서버의 취약점과 기존 방어선의 한계
포트 포워딩의 위험성: 공유기 포트를 외부에 개방하면 서버(집)의 실제 IP가 전 세계에 노출되어 디도스(DDoS) 공격의 표적이 된다.

소프트웨어 방어의 한계: 리버스 프록시나 방화벽(IPtables)만으로는 거대한 쓰레기 트래픽을 쏟아붓는 볼류메트릭 디도스 공격을 막을 수 없다. 물리적인 인터넷 회선 자체가 막혀버리기 때문이다.

<br>

## VPN(가상 사설망)의 원리와 필요성
작동 원리: 인터넷상에 암호화된 '비밀 터널'을 뚫어 데이터를 전송하는 기술이다. 이 과정을 통해 내 실제 IP를 숨기고 VPN 서버의 IP로 세탁할 수 있다.

보안 효과: 서버의 포트를 외부에 노출할 필요가 없어져 공격자의 타겟팅을 원천적으로 차단한다. 게임용으로는 속도가 빠르고 가벼운 WireGuard나 Tailscale이 권장된다.  

<br>

_VPN: Virtual Private Network_

<br>

## 클라우드 VPS를 활용한 하이브리드 방어 전략
클라우드 VPS란: 오라클, AWS 같은 대기업의 거대한 데이터 센터 자원을 쪼개어 원격으로 빌려 쓰는 가상 컴퓨터다.

디도스 방패 구축: 기본적으로 엔터프라이즈급 방어망이 적용된 클라우드 VPS를 앞단에 세워 플레이어의 접속을 받는다. 그리고 이 VPS와 실제 구동되는 내 집 컴퓨터를 VPN 터널로 연결해 안전하게 데이터를 주고받는 방식이다.

<br>

_VPS: Virtual Private Server_

<br>

## 오라클 클라우드 무료 티어의 보안과 데이터 관리
인프라 보안 vs 내부 보안: 오라클 측에서 물리적 장비와 기본적인 네트워크 방어는 책임지지만, 서버 내부의 OS 비밀번호나 애플리케이션 취약점 관리는 전적으로 서버 관리자의 몫이다.

<br>

**올바른 데이터 저장 방식**

단순한 서버 설정 파일은 txt, json 등으로 저장해도 무방하다.

유저의 중요한 정보나 대량의 데이터는 동시성 문제와 보안을 위해 **MySQL**이나 SQLite 같은 데이터베이스(DB)를 사용해야 한다. 민감한 정보(비밀번호, API 키 등)는 평문으로 저장하면 안 된다.

<br>

_궁극의 보안은 '백업': 오라클 무료 티어는 자원 부족 등의 이유로 경고 없이 서버가 삭제(회수)될 위험이 항상 존재한다. 따라서 크론탭(crontab) 등을 이용해 매일 데이터를 압축하고, 구글 드라이브나 개인 PC 등 **외부 공간으로 자동 백업(Off-site Backup)**하는 것이 가장 중요한 핵심이다._

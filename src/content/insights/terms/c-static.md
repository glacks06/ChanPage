## static 변수, 쉽게 말해 “평생 살아남는 변수”

보통 함수의 지역 변수는 함수가 끝나면 사라지지만,  
\`static\` 변수는 함수 안에 있어도 프로그램 시작부터 종료까지 쭉 살아남는다.  
이 변수들은 스택이 아니라 데이터(혹은 BSS, Initialized/Uninitialized Data Segment) 영역에 저장된다.

_※ C 프로그램 메모리 구조_  
_• 코드(Code) 영역: 함수 자체가 저장되는 곳._  
_• 데이터(Data) 영역: 전역 변수와 \`static\` 변수처럼, 프로그램 내내 살아있는 변수의 집._  
_• 스택(Stack) 영역: 함수 실행할 때마다 잠깐 만들어지는 공간. 함수 내 지역 변수, 매개변수 저장._  
_• 힙(Heap) 영역: 필요할 때 프로그래머가 직접 만드는 공간; 동적 할당._

<br>

## static 변수의 종류와 성격

**함수 내부 static 변수**  
• 프로그램 전체 생명주기동안 값 유지.  
• 함수가 여러 번 호출되더라도 초기화 구문(\`static int x = 0;\`)은 딱 한 번만 실행.  
• 접근 범위는 오직 그 함수 내부.

  
**전역 static 변수**  
• 역시 프로그램 전체 생명주기동안 값 유지.  
• 다만 해당 파일에서만 접근 가능. 파일 밖에서는 값이 안 보임.  
• 즉, “정보 숨기기”와 “평생 살아남기”가 동시에 된다.

<br>

## 실제 함수 호출 - 메모리 동작 이해하기

함수가 호출되면 스택에 “함수의 주소와, 복귀 주소”가 저장되고, 스택 프레임이 만들어진다.  
• 복귀 주소: 함수 끝나고 어디로 돌아갈지 알려주는 정보.  
• 함수 종료 시: 스택 프레임이 날아가고, 복귀 주소로 돌아가면서 지역변수도 소멸됨.  
• static 변수: 데이터 영역에 저장되어있는 변수라 함수 종료와 상관없이 계속 존재.  
C에는 “함수 인스턴스” 개념이 없고, 함수는 호출될 때마다 새 스택 프레임만 생긴다.

<br>

## static 변수의 초기화는 언제 될까?

• 컴파일러가 알아서 함수 실행과 독립적으로 처리함.  
• 초기화 코드는 프로그램이 실행될 때(로드 될 때) 데이터 영역에 저장된 값에 바로 반영됨.  
• 함수가 호출돼도 또 초기화하지 않음.  
즉, 함수 내부에서 static 변수 선언한다 해도  
실제 초기화는 프로그램 시작시 한 번만 진행.

<br>

## static 변수, 쓸 때 주의할 점

• 멀티스레드 환경에서 똑같은 static 변수에 여러 스레드가 동시에 접근하면 경쟁 조건(race condition)이 생긴다.  
• static 변수는 프로그램 끝날 때까지 메모리에 존재하니, 너무 많이 쓰면 메모리 낭비  
• 함수 내부/파일 내부 static 변수는 각자 다른 범위라 이름이 같아도 충돌 안 함.  
• 포인터나 참조(참고: C++에서)도 static으로 선언 가능. 생명주기 똑같아짐.

<br>

## 정리

static 변수는 “한 번 값이 만들어지면, 끝날 때까지 살아있는 변수”다.  
함수 내에 있든, 파일 전체에 있든 해당 범위에서만 보이고, 값은 프로그램 내내 유지된다.  
단, 메모리 관리와 호출 환경(특히 멀티스레드)에서는 항상 신경 써야 한다!

<br>

## 마무리

c언어에서 static 변수라는 것을 알게되어서 다른 언어에서 본 static 변수와 어떤 차이가 있을 지 궁금했다.

결론적으로 static 변수는 프로그램의 시작부터 끝까지 소멸되지 않고 남아있는 변수라는 것을 알게되었다.

다른 언어의 static 변수들과도 비교해보는 탐구도 진행하면 좋을 것 같다.

## 이중 포인터, 쉽게 말하면 “포인터를 가리키는 포인터”

일반 포인터는 변수의 메모리 주소를 저장한다면,  
이중 포인터는 다른 포인터 변수의 메모리 주소를 저장한다.

```c
int a = 10;
int *p = &a;        // p는 a의 주소를 저장
int **pp = &p;      // pp는 p의 주소를 저장
```

이렇게 되면 관계가 이렇게 된다:  
• \`a\`: 실제 값 10이 저장된 변수  
• \`p\`: a의 주소를 가리키는 포인터  
• \`pp\`: p의 주소를 가리키는 이중 포인터

<br>

## 값에 접근하는 방법

각각의 값에 접근하려면 역참조(\`\*\`) 연산자를 써야 한다.

```c
printf("a의 값: %d\n", a);          // 직접 접근: 10
printf("*p의 값: %d\n", *p);        // p를 통해 a에 접근: 10  
printf("**pp의 값: %d\n", **pp);    // pp를 통해 a에 접근: 10

printf("p의 주소: %p\n", p);        // a의 주소
printf("*pp의 값: %p\n", *pp);      // p의 값 (= a의 주소)
printf("pp의 값: %p\n", pp);        // p의 주소
```

여기서 중요한 건, \`\*\*pp\`는 별표 두 개를 써서 두 번 역참조 한다는 것이다.

_※ 포인터 변수 선언 시 작성한 '\*'기호의 개수가 마치 얼마나 타고 들어가야 주소가 아닌 값이 나오는지를 나타내는 듯하다._

<br>

## 언제 이중 포인터를 쓸까?

**1. 함수에서 포인터 변수 자체를 바꾸고 싶을 때**  
가장 흔한 용도다. 함수 안에서 포인터가 가리키는 값을 바꾸는 건 쉽지만,  
포인터 변수 자체의 주소를 바꾸려면 이중 포인터가 필요하다.

```c
void change_pointer(int **ptr) {
    static int new_value = 100;
    *ptr = &new_value;  // 포인터 변수가 다른 주소를 가리키도록 변경
}

int main() {
    int a = 10;
    int *p = &a;
    
    printf("변경 전: %d\n", *p);  // 10
    change_pointer(&p);           // p의 주소를 넘김
    printf("변경 후: %d\n", *p);  // 100
    
    return 0;
}
```

만약 \`change\_pointer(int \*ptr)\`로 했다면 포인터 변수 자체는 바뀌지 않고, 그냥 복사본만 바뀐다.

**2. 동적 2차원 배열 구현**  
2차원 배열을 동적으로 만들 때도 이중 포인터가 유용하다.

```c
int **matrix;
int rows = 3, cols = 4;

// 포인터 배열 할당 (각 행을 가리킬 포인터들)
matrix = (int**)malloc(rows * sizeof(int*));

// 각 행에 대한 메모리 할당
for(int i = 0; i < rows; i++) {
    matrix[i] = (int*)malloc(cols * sizeof(int));
}

// 이제 matrix[i][j]로 접근 가능
matrix = 1;
matrix[9][10] = 5;
```

## 포인터 타입이 중요한 이유

C 언어에서는 포인터가 가리키는 타입을 명확히 정의해야 한다.  
• \`int\*\`: \`int\` 타입 변수의 주소 저장  
• \`int\*\*\`: \`int\*\` 타입 변수의 주소 저장  
• \`int\*\*\*\`: \`int\*\*\` 타입 변수의 주소 저장  
따라서 이런 코드는 타입 불일치로 에러가 난다:

```c
int *p;
int *wrong_ptr = &p;  // 에러. int*의 주소를 int*에 저장할 수 없음

int **correct_ptr = &p;  // 올바름. int*의 주소를 int**에 저장
```

왜 이렇게 엄격할까? \`\*\` 연산자는 해당 주소에서 특정 타입 크기만큼 데이터를 읽으라는 명령이기 때문이다.  
• \`\*p\`는 “p가 가리키는 주소에서 int 크기(4바이트)만큼 읽어라”  
• \`\*\*pp\`는 “pp가 가리키는 주소에서 포인터 크기만큼 읽어서, 그 결과를 다시 주소로 해석해서 int 크기만큼 읽어라”  
타입이 안 맞으면 엉뚱한 크기로 데이터를 읽어버려서 프로그램이 망가질 수 있다.

## 마무리

C언어의 포인터 중 이중포인터에 대해서 한번 정리를 해봐야지 하다가 계속 미루게 되었다.

그래서 이번에 생각이 난 김에 정리를 했다.

결국 이중 포인터는 거창한 무언가가 아니라 포인터 변수의 주소를 저장하는 포인터 변수였다.

포인터의 작동 원리나 메모리에서의 작용들을 머릿속에 잘 이미지화하고 있다면 이해하지 못할 개념은 아닌 것 같다.

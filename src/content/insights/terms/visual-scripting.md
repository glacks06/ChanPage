## 비주얼 스크립팅의 개념
텍스트 코드를 직접 타이핑하는 대신, 화면상의 그래픽 요소(노드)와 이를 연결하는 선을 통해 프로그램의 논리 구조를 시각적으로 완성하는 프로그래밍 방식이다. 언리얼 엔진의 블루프린트, 유니티의 비주얼 스크립팅 등이 대표적이다.

<br>


## 주요 장단점 요약  

**장점**

낮은 진입 장벽: 코딩 문법을 몰라도 비개발 직군(기획, 아트 등)이 쉽게 다룰 수 있다.

직관적인 흐름: 데이터의 이동과 실행 과정이 시각적으로 명확하게 드러난다.

빠른 프로토타이핑: 즉각적인 아이디어 테스트와 구현에 유리하다.

문법 오류 방지: 오타로 인한 컴파일 에러가 원천적으로 차단된다.  

<br>


**단점**

스파게티 노드 현상: 로직이 복잡해질수록 선들이 엉켜 텍스트보다 가독성이 크게 떨어진다.

실행 성능 저하: C++, C# 등의 네이티브 텍스트 코드에 비해 연산 속도가 느리다.

버전 관리의 한계: Git 등을 이용한 다중 협업 시 코드 병합(Merge)과 충돌 해결이 까다롭다.

제어의 제약: 제공되는 노드에 의존해야 하므로 시스템 하단(Low-level)의 세밀한 제어가 어렵다.

<br>


## 성능이 더 느린 기술적 이유
텍스트 코드에 비해 CPU가 명령을 처리하기까지 중간에 거쳐야 하는 과정이 많기 때문이다.

인터프리터 오버헤드: 기계어로 즉시 실행되는 네이티브 코드와 달리, 가상 머신(VM)이 게임 실행 중에 실시간으로 노드를 읽고 해석(통역)해야 한다.

함수 호출 비용 증가: 단순 연산조차 무거운 객체(노드) 간의 데이터 전달과 함수 호출이라는 복잡한 과정을 거친다.

사전 최적화 부재: 현대 컴파일러가 제공하는 코드 압축 및 불필요한 연산 제거 등의 혜택을 받기 어렵다.

메모리 접근 비효율 (캐시 미스): 메모리상에 데이터가 흩어져 있어, CPU가 주소를 쫓아가며 데이터를 찾는 데 대기 시간이 발생한다.

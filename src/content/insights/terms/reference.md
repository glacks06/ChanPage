Reference는 '참조'라는 뜻이다. 

프로그래밍에서 Reference는 정확히 뭘까?

## Reference(참조)

Reference는 특정 변수, 객체, 메모리 주소 등을 참조하는 것이다.

여기서 참조라는 것은 일종의 또 다른 별명을 지정하는 것과 같다.

예를 들어보자.

당신의 이름은 김대학이다.

그리고 당신이 자주 하는 게임 속 닉네임은 '과제에찌든솔로'이다.

친구와 게임을 하는데, 친구가 당신을 '김대학'이라고 부르든 '과제에찌든솔로'라고 부르든 당신이 대답할 것이다.

'김대학'도 당신을 지칭하는 이름이고, '과제에찌든솔'도 당신을 지칭하는 별명이기 때문이다.

결국 이름은 달라도 그 이름이 가리키는 것은 같다.

참조도 이와 똑같다.

이를 코드상에서 보면

```
int value = 1000;
int& alias = value;
```

변수 value가 있고, alias가 value를 참조하는 형태를 예시로 볼 수 있다.

변수 value에 'alias'라는 별명이 붙었다.

alias와 value는 이름만 다를 뿐 같은 메모리 위치를 의미한다.

그래서

```
alias += 1000;
```

다음과 같은 연산을 하면 value의 값도 2000이 된다.

당연한 이야기다.

## Pointer(포인터) 변수와 Reference(참조) 변수의 차이

사실 포인터 변수와 참조 변수의 차이점에 대해서 조사를 해보면 언어에 따라 좀 다른것을 알 수 있다.

그래서 C/C++과 Javascript를 대표 예시로 조사하였다.

```
// Pointer
int value1 = 1;
int* p = &value1;

// Reference
int value2 = 1;
int& r = value2;
```

C++에서 Pointer와 Reference의 사용은 위와 같다.

p와 r은 각각 value1과 value2를 가리키기 때문에 p가 가리키는 위치의 데이터를 변화시키면 value1의 값도 변하고, 마찬가지로 r이 가리키는 위치의 데이터를 변화시키면 value2의 값도 변한다.

```
*p = 2; // value1값도 2으로 바뀜
r = 3; // value2값도 3으로 바뀜

printf("value1 = %d\n", value1);
printf("p = %d\n", *p);    
printf("value2 = %d\n", value2);
printf("r = %d\n", r);

/*

출력값:
value1 = 2
p = 2
value2 = 3
r = 3

*/
```

이렇게만 보면 포인터 변수와 참조 변수의 차이가 무엇인지 감도 안온다.

이 둘의 차이는 코드상이 아닌 개념적, 연산적 측면을 보아야 알 수 있다.

```
int value1 = 1;
int* p = &value1;
```

우선 (C/C++ 언어에서) 우리가 포인터 변수를 만들면 메모리에 주소값을 담을 공간을 할당받는다.

이 공간이 포인터 변수이며, 이 공간의 이름이 포인터 변수 이름(p)인 것이다.

위의 예시에서는 주소값을 얻어낼 수 있는 &기호를 이용해서 포인터 변수 p에 value1의 주소값을 저장한다.

_※이때 int옆에 \*을 붙임으로써 포인터 변수 타입임을 선언한다._

```
printf("%d", *p)
// 출력: 1
```

포인터 변수 옆에 '\*'기호를 사용하면 해당 변수에 주소값이 있으니 그 주소값으로 가라는 의미가 된다.

그래서 저 코드의 출력값은 value1에 담겨있는 값인 1이 된다.

이제 참조 변수를 살펴보자.

```
int value2 = 2;
int& r = value2;
```

참조 변수를 만들고 value2를 넣으면 value2의 공간에 r이라는 '별명'이 생긴다.

포인터 변수는 주소값을 담는 변수를 생성하고 그 공간에 주소값을 집어넣는 형식이였다면, 참조 변수는 어떠한 공간에 별명이다.

즉, r은 따로 공간을 생성하지 않는다.

그래서 포인터 변수에 '\*'기호를 붙여 저장된 주소값에 접근하던 방식과는 달리, r은 다른 기호를 붙이지 않아도 되는 것이다.

```
printf("%d", r);
// 출력: 2
```

여기서 개념적으로 차이가 생기는 것이다.

포인터 변수: 주소값을 담는 공간

참조 변수: 어떠한 공간의 또 다른 별명

※ 굳이 개념적 차이라고 한 이유는 포인터와 참조가 내부적인 연산이 같기 때문이다. 이 점은 포인터의 위험성과 참조의 필요성을 설명하면서 다시 보겠다.

포인터의 위험성과 참조의 필요성

포인터와 참조는 사용의 측면에서 매우 유사해서 왜 두개가 분리되어있는지 궁금하였다.

그 이유를 조사하면서 포인터의 위험성에 대해 알게되었다.

우선 포인터 변수는 주소값을 담는 변수이다.

그리고 특이한 점은 이 주소값에 어떠한 연산이 가능하다는 점이다.

예를 들어

```
#include <stdio.h>

int main()
{
    int numArr[5] = { 11, 22, 33, 44, 55 };
    int *numPtrA;
    int *numPtrB;
    int *numPtrC;

    numPtrA = numArr;    // 배열 첫 번째 요소의 메모리 주소를 포인터에 저장

    numPtrB = numPtrA + 1;    // 포인터 연산
    numPtrC = numPtrA + 2;    // 포인터 연산
    
    printf("%p\n", numPtrA);    // 00A3FC00: 메모리 주소. 컴퓨터마다, 실행할 때마다 달라짐
    printf("%p\n", numPtrB);    // 00A3FC04: sizeof(int) * 1이므로 numPtrA에서 4가 증가함
    printf("%p\n", numPtrC);    // 00A3FC08: sizeof(int) * 2이므로 numPtrB에서 8이 증가함

    return 0;
}
// 예제 출처
// https://dojang.io/mod/page/view.php?id=1925

// 출력 예시
// 00A3FC00 (메모리 주소. 컴퓨터마다, 실행할 때마다 달라짐)
// 00A3FC04
// 00A3FC08
```

위와 같은 코드에서 numPtrA는 배열의 첫번째 요소, numPtrB는 배열의 두번째 요소, numPtrC는 배열의 세번째 요소를 가리킨다.

+1 연산을 통해 다음 배열 요소로 주소 값을 변경시킨 것이다.

이를 '포인터 연산'이라고 한다.

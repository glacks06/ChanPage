<br>

Model Context Protocol(MCP)은 AI 모델이 외부 데이터 소스나 도구(Tool)에 접근할 때 사용하는 공통 규격이자 표준 인터페이스이다. 과거에는 가전제품마다 콘센트 모양이 달라 전용 어댑터가 필요했다면, MCP는 이를 USB-C 타입처럼 하나로 통일하려는 시도와 같다.

<br>

## 1. 해결하고자 하는 문제 (Fragmented Context)
기존에는 AI 어시스턴트가 사용자의 로컬 파일, GitHub 저장소, Google Drive, 데이터베이스 등에 접근하기 위해 각 서비스마다 별도의 API 연동 코드를 새로 작성해야 했다. 이 방식은 개발자가 새로운 도구를 추가할 때마다 유지보수가 어렵고, 보안 설정이 파편화되며 확장성이 떨어진다는 단점이 있었다.

<br>

## 2. 핵심 아키텍처
MCP는 크게 세 가지 구성 요소로 나뉜다.

* **MCP Host:** AI 모델을 구동하는 클라이언트 앱이다. (예: Claude Desktop, Cursor, IDE 등)
* **MCP Server:** 실제 데이터나 기능을 제공하는 프로그램이다. (예: 로컬 파일 시스템, Postgres DB, Slack API 등)
* **MCP Clients:** Host 내부에서 Server와 통신하며 모델이 이해할 수 있는 형태로 데이터를 전달하는 역할을 한다.

<br>

## 3. 주요 기능 (Capabilities)
MCP를 통해 AI는 다음과 같은 세 가지 방식으로 외부와 상호작용한다.

| 기능 | 설명 | 예시 |
| :--- | :--- | :--- |
| **Resources** | AI가 읽을 수 있는 정적 데이터 | 로컬 로그 파일, DB 스키마, 문서 내용 |
| **Tools** | AI가 직접 실행할 수 있는 동작 | 코드 실행, API 호출, 파일 수정 |
| **Prompts** | 미리 정의된 템플릿 | 특정 명령셋(예: "이 코드를 리뷰해줘") |

<br>

## 4. 중요성 및 전망
특히 개발 환경(IDE)에서의 활용도가 높다. 예를 들어 Cursor와 같은 에디터에 MCP를 설정하면 AI가 프로젝트의 파일 구조뿐만 아니라 연동된 DB의 테이블 구조를 실시간으로 파악하여 쿼리를 작성하고, 최신 문서를 참조하여 코딩을 돕는 것이 가능해진다. 

또한 데이터가 모델 학습에 무분별하게 사용되지 않고 로컬 서버를 통해 필요한 순간에만 전달되므로, 기업용 데이터를 활용할 때의 보안 우려를 낮출 수 있다는 장점이 있다.
